# README

## Epistemic Tool Functional Requirement Catalog

* **Conceptual Layer**

  * Provide minimal and precise definitions of the tool
  * Explicitly state the epistemic role and scope

* **Formal Layer**

  * Present the core mathematical structure
  * Allow symbolic and numerical inspection

* **Interactive Simulation**

  * Enable users to manipulate key parameters
  * Generate outputs dynamically (e.g., trajectories, distributions, events)

* **State Visualization**

  * Display the evolution of the system (e.g., intensity function, state variables)
  * Support multiple synchronized views

* **Stepwise Execution**

  * Decompose the mechanism into discrete steps
  * Allow users to traverse execution paths

* **Validation Module**

  * Implement diagnostic tests (e.g., residuals, invariants)
  * Show when and how the tool fails

* **Comparative Mode**

  * Allow side-by-side comparison with related tools
  * Highlight structural differences and trade-offs

* **Failure Exploration**

  * Let users intentionally violate assumptions
  * Observe resulting distortions in outputs

* **Artifact Inspection**

  * Expose the structure and interpretation of outputs
  * Enable downstream operations (prediction, sampling, inference)
